# 加密货币策略回测系统设计蓝图 (High-Level Blueprint)

> **[重要标注]**：本文件仅作为系统设计的 **High-Level 蓝图** 存储于全局 Skill 中，旨在提供架构参考和设计思路的持久化保存，**不直接用于自动化执行**。

## 1. 因子参数化
### 一种因子范式的执行，用户可以基于设定的参数组合生成对应的因子
- **变量包括**：元因子、统计流、横截面处理、定性判断等。
- **回测重点**：确保因子计算的向量化效率和历史数据的一致性。

## 2. 策略参数化
### 一种策略范式的执行，用户可以基于设定的参数组合生成对应的完整策略
- **变量包括**：回测时间段、持仓时间、选币个数、手续费率、滑点等。
- **回测重点**：模拟真实的交易环境，避免未来函数。

## 3. 策略多维度评估
### 策略评估的执行，自动化执行中非常重要的一环，代表了策略迭代的目标
- **核心指标**：夏普比率、最大回撤、卡玛比率、盈亏比、胜率、换手率等。

## 4. 策略自动迭代
### 自动化因子/策略寻参的执行，追求更高效更快速的策略迭代
- **常用算法**：随机测试、遗传算法、贝叶斯优化、网格搜索等。

---

## 5. 系统设计原则 (回测专用)
- **仅限回测**：本系统暂时与具体的策略实盘执行无关，不包含 API 下单逻辑。
- **模块化解耦**：因子生成、策略逻辑与评估系统应保持独立。
- **高性能要求**：针对 B 圈海量数据，优先使用 Pandas/Polars 等向量化工具。
