# 资金曲线指标计算 (Equity Curve Compute)

本 Skill 约定：在**默认《资金曲线》数据**（参见《crypto-knowledge-node》中《资金曲线》）之上，如何组织**常用指标**的输入、输出与计算逻辑。输出为一维 dict，便于引用与对接 §4.2 多维可视化。

## 1. 输入

- **数据来源**：《资金曲线》对应的 DataFrame 或文件路径。
  - 若为路径：通常为 `资金曲线.csv`，例如 SGT 数据目录下 `SGTs/{backtest_name}/{strategy_label}/资金曲线.csv`。
  - 若为 DataFrame：需包含《资金曲线》约定的核心列（至少含 `candle_begin_time`、`净值`、`净值max2here`、`净值dd2here` 等，见知识节点）。
- **时间范围**：可选。不指定时使用整段资金曲线；指定时仅在该时间区间内计算指标。

## 2. 输出结构

- **形态**：使用**一维表格**存储所有表现指标，便于后续引用与对接 §4.2「Expand 组合 × 指标」表格。
- **实现建议**：暂存于一个 **dict**，key 为指标名（str），value 为数值（float）；之后可按 key 引用具体指标、或转为 DataFrame 的一行。
- **key 命名**：
  - **整体指标**：`净值`、`最大回撤`、`回撤面积`、`净值回撤比`、`周度收益gmean` 等。
  - **分年度指标**：在指标名后加年份后缀，如 `净值_2022`、`最大回撤_2023`。年度由数据时间范围确定（见 §3.2）。

## 3. 指标计算逻辑

以下为当前约定的**计算公式与步骤**；实现时需保证输入为《资金曲线》DataFrame，且含列 `candle_begin_time`、`净值`、`净值max2here`、`净值dd2here`。

### 3.1 整体指标（基于整段曲线）

- **净值**  
  - 取《资金曲线》中**净值**列的**最后一个值**（期末净值）。

- **最大回撤**  
  - 取**净值dd2here**列的**最小值**（即回撤最大的一刻的回撤值）。

- **回撤面积**  
  - 对**净值dd2here**列直接**求和**（`sum`）。即把该列视为离散序列，不做归一化，积和为「回撤面积」。

- **净值回撤比**  
  - 公式：**净值**（即上述期末净值）除以 **回撤面积的绝对值**。  
  - 若回撤面积为 0，则需单独处理（如置为 `nan` 或无穷，由实现约定）。

- **周度收益几何平均**  
  - **第一步**：对**净值**列做 **rolling(168)** 的净值变化比例（168 小时 ≈ 1 周）。即每个时刻 t 的周收益为：  
    `ret_t = (净值_t / 净值_{t-168}) - 1`。  
  - **约束**：不使用 `min_periods`；若某时刻前不足 168 条，则该时刻的周收益为 **nan**，不参与后续计算。
  - **第二步**：对上述周收益序列（去掉 nan 后）取**简单版 gmean**：  
    - 取该序列的 **最小值、最大值、10%、25%、50%、75%、90% 分位数** 共 7 个代表值。  
    - 对这 7 个数做 **+1** 处理（避免负值无法求几何平均）：`x_i' = 1 + x_i`。  
    - 对 `x_i'` 求**几何平均**得到 `g`。  
    - **最终指标**：`周度收益gmean = g - 1`，即还原为围绕 0 的涨跌幅，可为负。

### 3.2 分时段指标：年度曲线指标

- **年份范围**：根据《资金曲线》的 **candle_begin_time** 确定涉及的自然年。例如数据从 2022 年 2 月 到 2024 年 3 月，则遍历 **2022、2023、2024** 三个年份，分别计算该年内的指标。
- **单年计算流程**（以 2022 年为例）：
  1. **截取**：按 `candle_begin_time` 筛出该年的所有行，得到该年的资金曲线子表。
  2. **净值归一化**：对该子表的**净值**列整体除以**该子表净值列的第一个值**，使该年段净值从 1 开始。
  3. **回撤数据**：与全量计算方式一致——基于**归一化后的净值**，按与《资金曲线》相同逻辑得到该年段内的 净值max2here、净值dd2here（即归一化净值的累计最大值、以及相对该最大值的回撤）。
  4. **指标计算**：在归一化净值与上述回撤列上，按 §3.1 的同一公式计算「净值、最大回撤、回撤面积、净值回撤比、周度收益gmean」。
  5. **命名**：每个指标名加上当前年份后缀，例如 `净值_2022`、`最大回撤_2022`、`回撤面积_2022`、`净值回撤比_2022`、`周度收益gmean_2022`。
- 各年指标一并放入同一 dict，与整体指标一起构成一维表（整体 key 无后缀，年度 key 带 `_YYYY`）。

## 4. 与上下游的衔接

- **上游**：依赖《资金曲线》的物理层与字段层定义（《crypto-knowledge-node》），以及回测落盘路径（如 SGT 数据目录）。
- **下游**：§4.2 SGTG 策略表现多维可视化中，「读取资金曲线并计算指标」一步应产出符合本 Skill 约定的指标结构；「Expand 组合 × 指标 表格」的列与本节指标名、分时段约定一致。

## 5. 后续可扩展

- 边界情况：无数据、单点、回撤面积为 0、周收益有效点不足 7 个时的处理约定。
- 更多指标：夏普比率、卡玛比率、胜率、换手率等可按同样 dict key 规范追加。
- 与《彩虹》或项目内现有指标模块的对接方式（若有）。

# 基于币池索引表的合约全市场资金曲线计算

- **数据源**：参考《crypto-knowledge-node》中的《币池索引表》，路径形如 `data/cache/all_factors_kline.pkl` 或 `data/cache/all_factors_kline.parquet`，包含 `candle_begin_time`、`symbol`、`is_spot`、`close`、`next_close` 等列。

- **工作流**：
  1. 筛选 `is_spot == 0` 的合约币种。
  2. 基于 `close` / `next_close` 在每一时间戳上计算全市场收益的**算术平均值**，得到按时间的单列序列，得到 2 列 DataFrame：`candle_begin_time`、`合约全市场下一周期收益`。
  3. 对 `合约全市场下一周期收益` 执行 `shift(1)`，偏移到下一周期，得到 `合约全市场当期收益`（即 t 时刻的当期收益 = 从 t-1 到 t 的收益率）。
  4. 对 `合约全市场当期收益` 执行 `fillna(0)`，至少填补第一周期的空值。
  5. 计算 `(1 + 合约全市场当期收益).cumprod()`，得到 `合约全市场模拟净值`。
   
- **输出**：DataFrame，包含 `candle_begin_time` 与 `合约全市场模拟净值` 两列。落盘或直接调用由调用方按需决定，此处仅阐明计算逻辑。

# 基于币池索引表的 BTC/ETH 资金曲线计算

- **数据源**：与「基于币池索引表的合约全市场资金曲线计算」相同，参考《crypto-knowledge-node》中的《币池索引表》。

- **工作流**：
  1. 筛选 `is_spot == 0` 的合约币种，**同时**筛选 `symbol in ['BTC-USDT', 'ETH-USDT']`。
  2. 在每个时间戳上，对 BTC-USDT、ETH-USDT 分别处理：**跳过算术平均值**，因每个时间戳每个 symbol 仅一行，直接取该行的 `close` / `next_close` 计算 `ret = next_close / close - 1` 即可。
  3. 对每个 symbol 的收益序列执行 `shift(1)`，得到当期收益（t 时刻 = t-1 到 t 的收益率），`fillna(0)`。
  4. 计算 `(1 + 当期收益).cumprod()`，得到各 symbol 的模拟净值。
  5. 输出列命名为 `BTC净值`（对应 BTC-USDT）、`ETH净值`（对应 ETH-USDT）。

- **输出**：DataFrame，包含 `candle_begin_time`、`BTC净值`、`ETH净值` 三列。落盘或直接调用由调用方按需决定，此处仅阐明计算逻辑。

# 资金曲线归一化

- **数据源**：DataFrame，包含 `candle_begin_time` 以及净值列（如 `净值`、`xx净值` 等，列名依具体数据而定）。

- **初始点归一化**：取各净值列的第一行数值，将各净值列除以该列的第一行数值，使起点归一为 1。

- **指定点归一化**：基于指定点 `(x, y)`，其中 `x` 为时间点、`y` 为指定净值。计算缩放比例：`scale = 输入 DataFrame 在时间 x 处的净值 / y`，然后将整列净值除以该缩放比例，使时间 x 处的净值归一为 y。

- **输出**：归一化后的 DataFrame。
