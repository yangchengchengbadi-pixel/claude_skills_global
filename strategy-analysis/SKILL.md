# 策略深度分析蓝图 (Strategy Analysis)

本蓝图定义生成 **`strategy_analysis.py`** 策略分析脚本的规范，该脚本采用**函数式编程**（非面向对象），用于对已完成回测的 SGT 进行深度分析，包括策略亏损时段、亏损贡献最大的币种及个币因子分析，支撑归因与优化。

**前置条件**：已存在可用的 SGT（或 SGTG 中的单个 SGT），且该 SGT 已完成回测、资金曲线已计算（即具备 SGT 数据目录、资金曲线、选币结果等）。

---

## 1. 核心定位

- **脚本定位**：`strategy_analysis.py` 是一个**函数式脚本**（非面向对象），包含用户输入区域和定制工作流函数。
- **结构设计**：
  - **用户输入区域**：脚本顶部设置用户输入变量（如 `strategy_label`），便于快速修改。
  - **工作流函数**：按步骤 2.0–2.4 实现对应的分析函数，按顺序调用执行。
- **功能定位**：策略深度分析是「生成与回测」之后的**分析层**：输入为 SGT 的 `strategy_label` 及其数据目录下的回测与选币结果，输出为亏损时段、亏损币种、个币因子可视化与因子分析结果，便于定位策略失效原因与改进方向。

---

## 2. 分析流程总览

| 步骤 | 名称 | 说明 |
|------|------|------|
| 2.0 | 用户输入与数据目录解析 | 用户输入 strategy_label，解析得到 SGT 数据目录，读取资金曲线、选币结果等数据文件 |
| 2.1 | 抓取策略亏损时间段 | （待填充） |
| 2.2 | 抓取亏损时间段内亏损最大的币 | （待填充） |
| 2.3 | 个币 k 线因子值可视化 | （待填充） |
| 2.4 | 个币因子分析 | （待填充） |

### 2.0 用户输入与全局数据准备

- **用户输入**：
  - `strategy_label`（字符串）：SGT 标识，例如 `"黄果树t3 -> meanbd0_168"`。
  - `top_n`（整数）：步骤 2.2 中，每个回撤区间统计的亏损最大 / 盈利最大 Top N 持仓段个数；用于整合 CSV 文件名及 Top N 筛选。
  - `single_coin_label`（字符串或 None）：指定单币持仓段标签（如 `GIGGLE-USDT_2025-10-22 16→2025-10-25 13`）。
    - **不为 None**：执行步骤 2.3 单币分析，执行完毕后退出，跳过 2.1、2.2。
    - **为 None**：跳过 2.3 单币分析，正常执行 2.1、2.2。
  - `factor_reference_lines`（list）：区域 C 因子值基准线，在右侧 Y 轴绘制灰色横线，如 `[0, 0.5, 1.0]`；空列表则不绘制。

- **SGT 数据目录解析**：
  - 根据 strategy-pipeline 蓝图约定，**SGT 数据目录**定义为 `SGTs/{backtest_name}/{strategy_label}`（双层结构）。
  - **解析方法**：
    1. 对输入的 `strategy_label` 执行 `strategy_label.split(' -> ', 1)`，得到 `[backtest_name, suffix]`（若 `strategy_label` 不含 ` -> `，则 `split` 结果长度为 1，此时 `backtest_name = strategy_label`）。
    2. 构建完整路径：`SGTs/{backtest_name}/{strategy_label}`。
  - **示例**：
    - 输入 `strategy_label = "黄果树t3 -> meanbd0_168"` → `split(' -> ', 1)` → `['黄果树t3', 'meanbd0_168']` → `backtest_name = '黄果树t3'` → 数据目录 = `SGTs/黄果树t3/黄果树t3 -> meanbd0_168`
    - 输入 `strategy_label = "黄果树t3"`（无后缀） → `split(' -> ', 1)` → `['黄果树t3']` → `backtest_name = '黄果树t3'` → 数据目录 = `SGTs/黄果树t3/黄果树t3`

- **全局数据文件准备**（按需读取）：
  - 在解析得到的 SGT 数据目录下，根据分析任务需要读取相应文件：
    - **资金曲线**（部分任务需要）：`资金曲线.csv`（包含 `candle_begin_time`、`净值`、`净值max2here`、`净值dd2here` 等列）
    - **选币结果**（部分任务需要）：`选币结果.pkl`（pkl 格式，包含各时间点的选币信息）
    - **回测config**（`config.pkl`，SGT 数据目录下）：《crypto-knowledge-node》中《回测config》的完整定义。策略分析中常用：`strategy_list`（建立策略名→持仓周期映射，与《选币结果》`strategy` 列一致）、`max_hold_period`（确定 data_start、回撤区间扩展范围等）、`factor_col_name_list`（回测config内所有因子参数组合）、各策略的 `long_factor_list` / `short_factor_list` / `long_filter_list` 等（提取前溯时间段，用于区域 C K 线截取）。
    - 其他回测结果文件（如 `策略评价.csv`、`资金曲线指标.csv` 等，按需读取）

### 2.1 抓取策略亏损时间段

- **数据源**：SGT 数据目录下的 `资金曲线.csv`（需包含 `candle_begin_time`、`净值`、`净值dd2here` 列）。

- **数据预处理**：
  - 确保 `candle_begin_time` 列转换为时间类型（如 `pd.to_datetime`）
  - 按 `candle_begin_time` 升序排序，确保时间顺序正确
  - 检查并处理缺失值：`净值dd2here` 为 `nan` 的行需跳过（使用 `dropna()` 或 `notna()` 筛选）

- **工作流**：
  1. **提取年份列表**：从 `资金曲线.csv` 的 `candle_begin_time` 列推断数据的时间范围，提取所有唯一年份（例如数据从 2022-03 到 2024-10，则遍历 2022、2023、2024 年）。
  
  2. **遍历每个年份**：对每个年份执行以下步骤：
     a. **找到最大回撤点**：
        - 筛选该年份的数据行（仅筛选时间范围，不重新计算任何列）
        - **重要**：`净值dd2here` 列直接使用全量资金曲线中的原始值，**不基于某一年重新计算**。因为新高到最大回撤的过程可能横跨年份，需要全量资金曲线的回撤值才能准确反映回撤幅度（例如：2022年12月达到新高，2023年2月达到最大回撤，如果只基于2023年重新计算会丢失跨年度信息）。
        - **边界情况**：如果该年份没有数据行（数据为空），跳过该年份
        - 找到 `净值dd2here` 值最小的时间点（即当年最大回撤点）
          - **注意**：`净值dd2here` 为负数，最小值对应最大回撤
          - **边界情况**：如果最小值 ≥ 0（该年份无回撤，一直在创新高），则跳过该年份或标记为"无回撤"
          - **多个相同最小值**：如果多个时间点的 `净值dd2here` 都是最小值，选择时间戳最大的（即回撤结束点）
        - 记录该时间点的 `candle_begin_time`（时间戳）和 `净值`
     
     b. **找到高点**（当年元旦至最大回撤点之间，净值最高的点）：
        - **搜索范围**：该年 **当年元旦 00:00** 至 **最大回撤点时间**（含）之间的所有数据行
        - **高点定义**：在上述范围内，取 **净值** 最大的那个时间点；不要求全历史新高，仅要求区间内最高。该点必然存在（至少最大回撤点自身在范围内）
        - 记录该高点的 `candle_begin_time`（时间戳）和 `净值`，以及所在年份（用于输出展示）

        - 计算区间的**回撤幅度**：`（高点净值 - 最大回撤点净值）/ 高点净值`（百分比；若为同一点则为 0%）

- **输出**：分两部分
  1. **文本输出**：不再使用 logger 打印各年份回撤区间详情（高点、最大回撤点、持续时间、回撤幅度）；仅当无任何回撤区间时输出提示。
  2. **资金曲线可视化**：复现彩虹默认回测图风格，使用 `core.figure.draw_equity_curve_plotly` 绘制主图（净值曲线 + 右轴回撤曲线），标题含累积净值、年化收益、最大回撤等指标，保存为 SGT 数据目录下的 `策略分析-资金曲线.html`；**画完后自动弹出该 HTML**（传 `show=True`，由 core 的 `show_without_plot_native_show` 打开）。
     - **主图**：始终绘制（净值 + 净值dd2here 右轴）。**每年最大回撤区间叠加**：根据 `find_max_drawdown_periods` 的返回，向 `draw_equity_curve_plotly` 传入 `drawdown_regions`；传入前将列表 **reverse**，使图例按年份升序。在主图上为每一年绘制半透明红色矩形（x 轴：高点 → 最大回撤点，y 轴上限：高点净值 `new_high_net_value`，y 轴下限：最大回撤点净值 `max_dd_net_value`），图例文案：`{年份}年：{高点月-日 时}→{回撤月-日 时}，{持续时长}，{回撤幅度}`（如 `2025年：05-11 14→05-13 12，3天5小时，-12.84%`）。
     - **子图执行逻辑**（不依赖 swap 与 BacktestConfig，不绘制 BTC/ETH 曲线）：
       - **子图**：仅当资金曲线 CSV 具备所需列且能获取杠杆时绘制；否则只出主图。
       - **仓位占比子图**：当 CSV 含 `equity`、`long_pos_value`、`short_pos_value` 时，从 `config.leverage` 读取杠杆（读不到则视为 1），计算 `long_pos_ratio`、`short_pos_ratio`、`empty_ratio` 及堆叠用 `long_cum`、`short_cum`、`empty_cum`，绘制多/空/空仓堆叠图。
       - **选币数量子图**：当 CSV 含 `symbol_long_num`、`symbol_short_num` 时，绘制多头选币数量与空头选币数量曲线。
       - **单笔持仓子图**：当 CSV 含 `long_max_ratio`、`short_max_ratio_abs`（及可选 `top3_long`、`top3_short`）时，绘制多头/空头单币最大持仓（百分比）及悬停前几名信息。
       - 仅当**同时满足**「有仓位占比所需列 + 有选币数量列」时设置 `show_subplots=True`，否则 `show_subplots=False`，仅保存主图。

### 2.2 抓取亏损时间段内亏损最大的币

- **核心目标**：遍历步骤 2.1 输出的**每一年的最大回撤区间**，针对每个区间，计算每个币种对策略净值下跌的**实际贡献度**，找出该区间内亏损最大的“罪魁祸首”。（以下工作流为针对**单个回撤区间**的执行逻辑，需在遍历循环中调用。）

- **整体流程（遍历前/后）**：
  - **进入回撤区间遍历前**：初始化一个用于汇总「所有回撤区间 Top N 单币持仓段」的 DataFrame（或列表）。遍历时，每个区间将本区间的**亏损最大 Top N** 与 **盈利最大 Top N** 持仓段行数据追加到该汇总中；**不再**对各区间做 Top N 的 logger 逐条输出。
  - **所有回撤区间分析结束后**：将上述汇总 DataFrame 先按 **`回撤区间分析目录名`、`pnl_contribution` 升序排序**，再保存至 **SGT 数据目录**（即各区间分析子目录的上一级），文件名为 **`回撤区间分析整合_top{x}单币持仓段表现.csv`**，其中 `x` 为 Top N 的 N（如 `top_n=3` 则文件名为 `回撤区间分析整合_top3单币持仓段表现.csv`）。

- **数据源**：
  1. **选币动作**：SGT 数据目录下的 **《选币结果》**（`选币结果.pkl`，含 `candle_begin_time`, `symbol`, `strategy`, `target_alloc_ratio`）。**性能**：在遍历各回撤区间前读入一次并做好列处理（symbol、candle_begin_time 去时区），遍历时对每个区间仅做时间切片并复用同一内存变量，避免每区间重复读盘。
  2. **持仓配置**：SGT 数据目录下的 **《回测config》**（`config.pkl`，**权威源头**）。需读取其中的 `strategy_list`（由多个 **《单策略 dict》** 解析而成的 `StrategyConfig` 对象），建立 `{策略名: 持仓周期(小时数)}` 的映射字典（如 `{'#0.xxx': 12}`）。**策略名对应关系**：框架在写入《选币结果》时，`strategy` 列取值为 `stg_conf.name`（见 `core/select_coin.py`），因此映射 key 必须用 **`stg.name`**，与《选币结果》的 `strategy` 列一致；持仓周期取 **`stg.period_num`**（如 `hold_period='12H'` → 12），与逐小时时间对齐。*注：单策略 pkl 的文件名使用 `get_fullname(as_folder_name=True)`，与列值 `stg.name` 不同，建立映射时以列值为准。*
  3. **逐时行情**：项目缓存目录下的 **《币池索引表》**（`data/cache/all_factors_kline.parquet` 或 `.pkl`）。提供全市场、全时段的 `close` 价格，用于计算区间内每一小时的涨跌幅（**选币端**个币的收益率矩阵）。
  4. **全市场基准（仅中性策略）**：SGT 数据目录下、**文件名包含“合约全市场”**的单策略 pkl（结构与《选币结果》相同，含 `candle_begin_time`, `symbol`, `close`, `next_close`）。该文件为未对冲的合约全市场币池，用于计算全市场平均收益作为基准，避免使用《币池索引表》时新币未过滤的问题。选用哪个文件与《选币结果》中包含的策略一致：若包含 `'多合约全市场'` 则读取**文件名包含“多合约全市场”**的 pkl；若包含 `'空合约全市场'` 则读取**文件名包含“空合约全市场”**的 pkl。

- **工作流**：
  1. **数据加载与对齐**：
     - 读取 **《回测config》**，建立 **`{策略名: 持仓周期(小时数)}`**：策略名以《选币结果》的 `strategy` 列为准，与 `strategy_list` 中 `StrategyConfig` 的 **`stg.name`** 一致（框架落盘时写入的即为 `stg.name`）；持仓周期取 **`stg.period_num`**（如 `hold_period='12H'` → 12 小时），与逐小时时间对齐。
     - 确定回撤区间 `[T_start, T_end]`（来自步骤 2.1）。**矩阵与数据的时间窗口**统一为 **`[data_start, T_end]`**，其中 `data_start = T_start - max_hold_period`（从《回测config》读取 `max_hold_period` 并换算为时间，如 12H→12 小时），以覆盖区间初期已存在的持仓并在每个时间戳上对齐仓位与收益。
     - **创建分析子目录**：在 SGT 数据目录下创建子文件夹，命名格式为 `回撤区间分析_{T_start}→{T_end}`，时间含日与小时且**不含冒号**（如 `回撤区间分析_2022-01-01 00→2022-02-02 12`）。后续所有落盘文件均存放于此。
     - 加载 **《选币结果》** 和 **《币池索引表》**，并按时间窗口 `[data_start, T_end]` 筛选。
     - **数据落盘**：
       - 将 **筛选后选币结果** 保存至分析子目录：`选币结果_filtered.pkl`（pickle，落盘比 csv 快；可选由参数控制是否落盘）。
     - **中性策略预处理**：检查《选币结果》的 `strategy` 列。
       - 若包含 `'多合约全市场'`，标记当前分析模式为 `'neutral_short'`（中性做空），并从数据中**剔除** `'多合约全市场'` 对应的所有选币记录（对冲端不参与个币归因）；同时确定基准数据源为**文件名包含“多合约全市场”**的 pkl。
       - 若包含 `'空合约全市场'`，标记当前分析模式为 `'neutral_long'`（中性做多），并从数据中**剔除** `'空合约全市场'` 对应的所有选币记录；同时确定基准数据源为**文件名包含“空合约全市场”**的 pkl。
       - **全市场基准序列**：在 SGT 数据目录下读取上述确定的合约全市场 pkl（单策略、未对冲），基于其 `close` / `next_close` 在每一时间戳上计算全市场收益的**算术平均值**，得到按时间的单列序列。计算完全量基准后，先保存至 **SGT 数据目录**，文件名为 **`收益率矩阵_策略内合约全市场_全量.pkl`**；再截取时间窗口 **`[data_start, T_end]`** 内的数据，保存至分析子目录，文件名为 **`收益率矩阵_策略内合约全市场.pkl`**。若某时刻无数据则保留 NaN，不填充。
       - 目的：聚焦于 Alpha 端（选币策略）的亏损归因，排除 Beta 对冲端的影响；基准使用与回测一致的、新币过滤后的合约全市场币池。

  2. **构建持仓矩阵 (Position Matrix Reconstruction)**：
    - **核心概念（向量化）**：将《选币结果》中的 **(时间, symbol, 权重)** 转为 **(行, 列, 值)**，校验时间与 symbol 是否在矩阵预设范围内，再用 **`np.add.at`** 批量把权重加到差分矩阵的对应格子上（开仓 +w、平仓 -w），最后沿时间 **cumsum** 得到持仓矩阵。避免逐行循环，提升性能。
    - **策略遍历**：对时间窗口筛选 + 对冲策略剔除后的《选币结果》，读取 `strategy` 列的所有**独特值**，录入参考变量 **「所有单策略」**（该独特值集合，供后续引用）；遍历每个策略，基于该策略生成对应的持仓矩阵，服务之后的单币分析。
    - **二次筛选**：对每个策略，从上述选币结果中进一步筛选出 `strategy == 当前策略` 的行，得到**二次筛选后的选币结果**。
    - **初始化全零矩阵**（针对每个策略）：
      - **行索引（时间）**：基于时间窗口 **`[data_start, T_end]`**（即 `T_start - max_hold_period` 至 `T_end`）生成的逐小时时间序列，时间窗口不变。
      - **列索引（币种）**：基于**二次筛选后的选币结果**中出现过的所有 `symbol` 唯一值。
    - **基于选币生成动态持仓（差分还原法）**（针对每个策略）：
      - 所有计算均基于**二次筛选后的选币结果**。
      - **第一步：录入开平仓信号（Signal Injection）**
        - 将选币表的「时间、symbol、权重」整列转为等长的「行号、列号、要加的值」：时间用 `time_index.get_indexer` 得行号，symbol 用 `symbol_to_col` 得列号；校验行号 ≥0、symbol 在列索引内，过滤无效记录。
        - **开仓**：在差分矩阵的 (行, 列) 处累加 `+w`；**平仓**：在 (t+h 对应行, 列) 处累加 `-w`。用 `np.add.at` 一次性写入，同一格多次开/平仓会累加/累减。
        - *注意*：若 `t+h` 超出矩阵时间范围，则不录入该平仓信号，区间末尾该币种仍处于持仓状态。
        - *示例*：T=0 买入 BTC 权重 0.1、持仓 3h → 行 0 列 BTC +0.1，行 3 列 BTC -0.1。
      - **第二步：生成实时持仓比例（Position Generation）**
        - 对差分矩阵沿时间轴（行方向）执行 **累积求和 (CumSum)**，得到每一时刻、每个币种的**实际持仓权重**（自动处理多笔错峰叠加）。
      - **数据落盘**：将 **持仓矩阵** 保存至分析子目录，文件名为 **`持仓矩阵_{strategy}.pkl`**（`strategy` 为当前策略名）；同时保留 **`持仓矩阵.pkl`**（合并所有策略，供后续贡献矩阵与单币持仓段使用）。

  3. **计算收益率矩阵 (Return Matrix Calculation)**：
     - **初始化**：创建一个与持仓矩阵**维度完全一致**（行=时间，列=币种）的矩阵。
     - **基础收益率（绝对）**：基于《币池索引表》计算全市场逐时收益率。
       - 采用 **Forward Return** 逻辑：`Ret(t, s) = Next_Close(t, s) / Close(t, s) - 1`。
       - **对齐方式（性能）**：将《币池索引表》按时间窗口 **`[data_start, T_end]`** 筛选后，用 **pivot**（index=时间, columns=币种, values=ret）得到「时间×币种」表，再按与持仓矩阵一致的逐小时时间索引与币种列 **reindex** 一次得到收益率矩阵；缺数保留 NaN。避免按币、按时间双重 Python 循环，以降低长区间耗时。
       - *核心逻辑*：
         - **时间对齐**：`Ret(t)` 计算的是从 `t` 时刻（Close）到 `t+1` 时刻（Next_Close）的收益率，即 `t` 时刻 K 线收盘后持仓一个周期的收益。
         - **矩阵一致性**：这与持仓矩阵中 `Position(t)` 代表“在 `t` 时刻产生的持仓信号（并于 `t+1` 周期实际持仓）”完全对齐。
         - **无需 Shift**：因此，持仓矩阵与收益率矩阵**无需任何 Shift 操作**，直接对应元素相乘即可得到正确的盈亏贡献。
     - **中性策略超额处理**：
       - 若当前模式为 `neutral_short` 或 `neutral_long`：
         - **全市场基准收益**：使用步骤 1 中由「合约全市场」单策略 pkl 计算并已落盘的基准序列 `Benchmark_Ret(t)`（即 `收益率矩阵_策略内合约全市场.pkl`），不再使用《币池索引表》计算基准。
         - 计算 **超额收益 (Alpha)**：`Alpha_Ret(t, s) = Ret(t, s) - Benchmark_Ret(t)`。若某时刻基准为 NaN，则对应超额为 NaN，不填充。
         - **数据落盘**：分别保存 `收益率矩阵_绝对.pkl`、`收益率矩阵_策略内合约全市场.pkl`（步骤 1 已落盘）、`收益率矩阵_超额.pkl` 至分析子目录，便于快速读写大矩阵。
         - 使用 `Alpha_Ret` 作为后续计算贡献的最终收益率矩阵。
       - 若为非中性策略（如纯多头/纯空头）：
         - **数据落盘**：保存 `收益率矩阵_绝对.pkl` 至分析子目录。
         - 直接使用基础 `Ret` 作为后续计算贡献的最终收益率矩阵。
     - *注：收益率矩阵仅反映价格变化（或相对变化），不包含方向信息；方向将由持仓矩阵中的权重正负（Step 4）自动处理。*

  4. **计算逐时盈亏贡献与聚合 (Contribution & Aggregation)**：
     - **贡献矩阵**：`Contrib(t, s) = Position(t, s) * Final_Ret(t, s)`。
       - **计算方式**：**元素级相乘 (Element-wise Multiplication / Hadamard Product)**。即矩阵中对应位置的元素直接相乘，而非矩阵乘法（Dot Product）。
       - `Position(t, s)`：来自步骤 2 的实时持仓权重（含正负方向）。
       - `Final_Ret(t, s)`：来自步骤 3 的最终收益率（绝对收益或超额收益）。
     - **数据落盘**：
       - 将 **贡献矩阵** 保存至分析子目录（`贡献矩阵.pkl`），便于快速读写大矩阵。
     - **单币持仓段分段与统计**：
       - **定义持仓段**：针对每个币种，将其在 `Position` 矩阵中的时间段切分为“持仓段”（Segment）。采用 **连续同号非零**：同一段内权重符号不变，多空分开统计；若先多后空或先空后多，则拆成两段；中间出现平仓（Position=0）也视为段边界。
       - **过滤持仓段**：只保留 **开始时间 ≥ T_start** 的持仓段。因 [data_start, T_start) 未录入区间前带入持仓，该段内总体持仓数据不全，不参与归因统计与 Top N 输出。
      - **统计持仓段表现**：
        - 遍历过滤后的所有持仓段。
        - 计算每个段的 **累积盈亏贡献**：对该段内 `Contrib(t, s)` **仅对非 NaN 元素求和**（如 `np.nansum` 或 `sum(skipna=True)`）。这样在币下架导致部分时刻缺数时，不虚构 0，只汇总已知贡献，逻辑更严密。
        - 记录段的 `开始时间`、`结束时间`、`持续时长`、`最大持仓权重`；并记录 **`n_valid_periods`**（该段内有效贡献的期数）、**`n_missing_periods`**（该段内 NaN 的期数），便于判断是否有下架/缺数导致的统计不完整。
      - **生成持仓段报表**：
        - 构建单币持仓段报表，包含列：`symbol`, `start_time`, `end_time`, `duration`, `pnl_contribution`, `max_weight`, `n_valid_periods`, `n_missing_periods`，以及两列标记信息：
          - **回撤区间分析目录名**：当前持仓段所属的回撤区间分析目录名，例如 `回撤区间分析_2023-01-02 03→2023-05-02 22`。
          - **单币持仓段标签**：由 `symbol` 与时间段拼接，例如 `GIGGLE-USDT_2025-10-22 16→2025-10-25 13`，用于唯一标识该持仓段。
        - 将该报表保存至**分析子目录**：**`单币持仓段表现.pkl`**（pkl 落盘，加速；不再使用 csv）。
        - 按 `pnl_contribution` 排序后，取**亏损最大 Top N** 与 **盈利最大 Top N** 的行数据，**追加到调用方传入的汇总 DataFrame** 中，供遍历结束后统一落盘；**不再**对本区间做 Top N 的 logger 逐条输出。

- **输出**：
  - **中间过程文件**（各回撤区间分析子目录下）：`选币结果_filtered.pkl`（可选）, `持仓矩阵.pkl`（合并所有策略）, **`持仓矩阵_{strategy}.pkl`**（按策略分拆，服务单币分析）, `收益率矩阵_绝对.pkl` / `收益率矩阵_策略内合约全市场.pkl` / `收益率矩阵_超额.pkl`, `贡献矩阵.pkl`, **`单币持仓段表现.pkl`**（全量持仓段，供核查）。
  - **SGT 数据目录下**（中性策略时）：**`收益率矩阵_策略内合约全市场_全量.pkl`**（全量全市场基准序列）。
  - **分析结果文件**：
    - 各子目录：`单币持仓段表现.pkl`（见上）。
    - **SGT 数据目录下**：**`回撤区间分析整合_top{x}单币持仓段表现.csv`**（在所有回撤区间的 Top N 亏损 + Top N 盈利持仓段汇总后，按 `回撤区间分析目录名` 和 `pnl_contribution` 升序排序得到，x 为 top_n 的数值）。写入整合 CSV 后，可选地做一次 logger 输出：
      - 遍历整合表中的每一个 **`回撤区间分析目录名`** 分组；
      - 对每个分组，逐行输出其 **`单币持仓段标签`**、对应的 `pnl_contribution` 以及 `max_weight`（二者均以百分比格式，保留 2 位小数，如 `3.45%`），便于在日志中快速浏览各回撤区间的关键持仓段。

- **区域 C**，单币持仓段主图：
  - **位置**：子图矩阵的第一行第二列（右上）。左侧 Y 轴为价格，右侧 Y 轴为因子数据。与区域 D **X 轴联动**（见区域 D 的「子图 X 轴联动规则」）。

  - **《因子数据》合并约定**：按数据格式区分。**parquet**：由 pkl 转换时已统一排序 `['candle_begin_time', 'symbol', 'is_spot']`，按**行位置**合并（`df[col] = df_factor[col].values`），与 core_2 一致。**pkl**：来自 Core，按 **index 对齐** 合并（`df[col] = df_factor`）。读取时优先 parquet，fallback 到 pkl。
  - **Y 轴左侧工作流**：
    1. **解析 symbol**：从 `single_coin_label` 按下划线 `_` 分割，取**第一段**作为 symbol（如 `GIGGLE-USDT_2025-10-22 16→2025-10-25 13` → symbol 为 `GIGGLE-USDT`）。
    2. **读取 K 线 df**：从《全量K线list》中遍历，找到首个 `df['symbol'].iloc[0] == symbol` 的 DataFrame；若未找到则跳过区域 C。记录该 df 第一根 K 线的时间点，称为**K线初始时间点**。
    3. **确定前溯时间段**：在 SGT 数据目录下读取《回测config》（`config.pkl`），遍历 `conf.strategy_list` 中每个策略的以下列表，提取其中所有元组的 `param` 参数：`long_factor_list`、`short_factor_list`（对应 factor_list）、`long_filter_list`、`short_filter_list`（对应 filter_list）、`long_filter_list_post`、`short_filter_list_post`（对应 filter_list_post）。对每个 `param`：若为 `int` 则取该值，若为 `tuple` 则取 `max(param)`；最终取所有收集值的**最大值**作为前溯时间段（单位：小时，与 1h K 线一致）。

### 2.4 个币因子分析

- **核心变量**
  - **策略亏损时间表**
    - **定义（目标）**：基于选币时间的策略亏损量。针对**当前单币持仓段**（指定币对 + 对应持仓段时间范围），刻画该持仓段内每个选币时间点在未来一个持仓周期内的亏损程度，为后续与因子值做对应计算、挖掘因子与策略亏损的关系提供标签。**范围限定**：仅限当前指定币对及该持仓段；**计算规则**：持仓段内每个时间点 t 对应的「未来一个持仓周期的收益贡献」，若为正则置 0，否则保留负值，即为该时间点的策略亏损量（策略亏损时间表中的一项）。
    - **数据源**：《回测config》；《贡献矩阵》（回撤区间分析目录下 `贡献矩阵.pkl`）；当前单币持仓段标签 `single_coin_label`（解析得到指定币对 symbol 与持仓段时间范围 [seg_start, seg_end]，即区域 C 第 4 步的时间范围）。
    - **工作流**：
      1. **确认持仓周期**：从《回测config》中抓取所有**不带有「合约全市场」**的多空策略，读取各自的 `hold_period`（如 `12H` → 12）。若这些 `hold_period` 中存在超过一个独特值，则**直接报错退出**，不支持分析多个持仓周期。否则取该唯一值作为持仓周期 N（单位：小时，与 1h K 线一致）。
      2. **读取并截取贡献矩阵**：在回撤区间分析目录下读取 `贡献矩阵.pkl`；按当前时间范围及当前 symbol 截取单列数据。**不做 shift(1)**（与区域 D 其他曲线不同）：此处刻画的是未来持仓周期的收益，而非当下时间戳的收益。
      3. **计算选币点未来持仓收益（前向均值）**：对截取的单列数据，每一行计算「当前时间向后延伸 N 个时间戳」的即时贡献的**算术平均**（包括当前时间戳的值）。即：对时间 t，取 `mean(contribution[t], contribution[t+1], …, contribution[t+N-1])`。注意：这不是 rolling（向后看），而是 rolling 的反方向——向前延伸。**范围裁剪**：只保留 `[seg_start, seg_end - (hold_period - 1)]` 范围内的数据，该范围外的值用 0 填充。原因：当 t > seg_end - (N-1) 时，前向窗口会超出持仓段终点，无法得到完整 N 期前向收益，故视为无效选币点。*备注*：用算术平均来当做亏损的加权是为了方便计算，涨跌幅较大时不精确，仅供参考。
      4. **得到策略亏损时间表**：对步骤 3 的结果，若 > 0 则置 0，否则保留负值，得到策略亏损时间表（本地变量）。
      5. **绘制到区域 D**：将策略亏损时间表在区域 D 的**右侧 y 轴**绘制，图例命名为「基于选币点的策略亏损权重」，使用 `fill='tozeroy'` 填充至零线，**默认展示**（不设 `visible='legendonly'`）。
    - **输出**：策略亏损时间表（本地变量）+ 区域 D 的绘图更新。
  - **因子排名亏损加权平均值**
    - **定义**：当前多空策略内的选币/过滤因子排名，基于策略亏损的加权平均，刻画了当前单币持仓段内、亏损时因子排名的典型值。

- **复制 SGT 输入逻辑**：将 SGT 目录下的 `SGT输入逻辑.md` 直接复制到单币持仓段目录下。

- **策略因子计算**
  - **Note（脚本实现）**：为复用数据加载，脚本中将「绘制因子数据」与「策略因子计算」合并到同一 for 循环，tqdm 标签为「因子计算&绘制」；蓝图逻辑仍分开描述。
  - **数据源**：区域 C 计算的因子数据（各因子的原始值列与 `{factor_name}_pct_rank` 排名列）；策略亏损时间表。
  - **范围**：`[seg_start, seg_end - (hold_period - 1)]`（与策略亏损时间表有效范围一致）。
  - **工作流**：对每个策略因子，依次计算以下 4 项：
    1. **策略因子值区间平均**：范围内因子原始值的算术平均。
    2. **策略因子值亏损加权平均**：`Σ(因子值 × 策略亏损时间表) / Σ(策略亏损时间表)`。
    3. **策略因子排名区间平均**：范围内因子排名（`{factor_name}_pct_rank`）的算术平均。
    4. **策略因子排名亏损加权平均**：`Σ(因子排名 × 策略亏损时间表) / Σ(策略亏损时间表)`。
  - **输出**：将每个策略因子的 4 项计算结果保存到单币持仓段目录下的 `因子计算结果.md`。

- **额外因子计算**
  - **Note（脚本实现）**：同上，与「绘制额外因子」合并到同一 for 循环，tqdm 标签为「因子计算&绘制」。
  - **数据源**：区域 C 右侧 y 轴绘制的额外因子数据（data/cache 中排除 `factor_col_name_list` 的因子，各因子的原始值列与 `{factor_name}_pct_rank` 排名列）；策略亏损时间表。
  - **范围**：`[seg_start, seg_end - (hold_period - 1)]`（与策略因子计算一致）。
  - **工作流**：与策略因子计算完全相同，对每个额外因子依次计算上述 4 项。
  - **输出**：将每个额外因子的 4 项计算结果追加到同一 `因子计算结果.md` 文件中。

- **因子数据整理**（写入 md 前执行）
  - **因子参数组合**：表格首列命名为「因子参数组合」，每行对应一个因子参数组合（如 `TpNMeanbd0_168`）。
  - **因子多参数混合结果**：在策略因子、额外因子各自范围内，对相同因子名称的不同因子参数组合做整合；提取因子名称规则：若参数组合以 `_数字` 结尾则因子名为其前缀，否则整体为因子名；对整合后的多行计算 4 项指标的**算术平均**，作为该因子名称的汇总行。按「策略因子排名亏损加权平均」**从小到大**排序（排名越小越靠前）。
  - **因子参数组合明细排序**：按因子多参数混合结果的排序次序展示，同一因子名称的参数组合集中展示（如 TpNMeanbias1 的 _168、_300、_600 连续排列）。
  - **排名百分比格式**：因子排名相关列（策略因子排名区间平均、策略因子排名亏损加权平均）以百分比格式展示（如 `0.1559` → `15.59%`）。
