# 策略全周期开发工作流 (Strategy Full Cycle Workflow)


## 工作流步骤总览

| 步骤编号 | 名称 | 说明 |
|----------|------|------|
| 101 | 额外因子压力测试 | 因子库准备、AI 补全 13 位密码、数据生产与特征统计 |
| 102 | AI 生成策略优化方案 | 多模态诊断、备选前滤方案、落盘保存 |







## §101 额外因子压力测试 (Factor Stress Test)

### 101.1 额外因子库准备 (Library Preparation)
- **步骤 A：用户意图录入**：用户在 `strategy_iter_data.py` 的 `dict_factor_fg13bits` 中填入想要的因子名称 (Key)，Value 留空。
- **步骤 B：AI 补全13位密码 (AI Auto-Completion)**：
  - **推荐提示词**（将下方提示词发送给 AI）：
```
sm，请参考 `factor_generator.py` 中定义的 13 位因子流水线范式（包含 s1-s13 每一位的物理意义、计算流模板及命名规则），补全 `strategy_iter_data.py` 中 `dict_factor_fg13bits` 缺失的 13 位密码列表（Value）。请确保设计的密码在逻辑上与 Key 名高度吻合，且在执行脚本时能产生 `✅ [MATCH]` 的校验结果。
```
- **步骤 C：自动化生成与校验 (Automated Logic)**：
  - **执行**：用户运行 `python strategy_iter_data.py`。
  - **校验**：系统自动调用 `fg.generate()` 生成物理脚本（`factors/` 或 `sections/`），并严格校验 `Key` 与生成的 `actual_name` 是否吻合。
  - **溯源**：Log 需显式展示 XS 因子对应的 Base TS 因子名称，确保逻辑链条清晰。

### 101.2 数据生产与特征统计
- **步骤 A：生产**：用户运行 `python strategy_gt.py`（强制计算 `l_custom_factors`）。
- **步骤 B：统计**：用户运行 `python strategy_analysis.py`（计算算术平均 vs 亏损加权平均）。单币持仓段目录下会生成 `因子计算结果.md` 以及其他默认的初步分析结果。

## §102 AI 生成策略优化方案 (AI Prescription)

### 102.1 中性策略优化 (Neutral Mode)
*适用场景：`execution_mode` 为 `neutral_short/long`。核心在于保持对冲平衡。*

**推荐提示词**（将下方提示词 + 单币持仓段目录路径一并发送给 AI；可将 `{单币持仓段目录路径}` 替换为实际路径，或使用 @ 引用该目录）：

```
sm，请对以下单币持仓段目录执行策略亏损诊断与优化方案生成（strategy_full_cycle_workflow §102.1 步骤 A/B/C/D）：

【分析数据源】{单币持仓段目录路径}

请按以下结构输出文本：
1. **主观分析**：基于 因子计算结果.md 与目录下的可视化图表，阐述当前亏损的归因（异常因子、K 线形态、市场环境等）。
2. **备选前滤方案**：基于强信号识别与逻辑推演，以文本形式给出 Top 3 备选前滤方案（如「QvNEwm3kNRank 排名前滤 90% 以上的」）；若有效选项不足可减少至 1-2 个，或说明未找到合理方案。暂不需要具体过滤元组。
3. **其他观察**：（可选）风险提示、需进一步验证的假设等。

请将上述完整输出保存至【分析数据源】目录下，文件名为 `ai分析以及优化方案.md`。
```

- **步骤 A：多模态联合诊断**：
  - **参考 A**：`因子计算结果.md` 中的算术平均 vs 亏损加权平均（抓取异常 Delta）。
  - **参考 B**：目录下的 `.png` 视觉图表（分析 K 线形态、归一化市场相对强度、实际持仓亏损节奏）。
- **步骤 B：额外前置过滤 (Pre-Filter)**：
  - **逻辑**：基于异常因子值设计过滤规则（注：中性策略原则上严禁使用后置过滤）。
  - **强信号识别**：优先观察「策略因子排名亏损加权平均」。若某因子的多参数混合结果中该指标**极端**（接近 0% 或 100%），说明该因子信号较强；若该因子多个参数下的表现**普遍极端**，为加分项；若「策略因子排名区间平均」不极端但「策略因子排名亏损加权平均」极端（Delta 大），亦为加分项。「策略因子值」带量纲，一般不直接用因子值做前滤（比例会不稳定）；若发现特别异常、违反常识的值，请在主观分析中阐述。
  - **输出**：以**文本**形式提供 **Top 3** 备选前滤方案，尽量提出 3 个；若实在找不到有效选项，可减少至 1-2 个，甚至明确说明「未找到合理的备选前滤方案」。暂不需要输出具体的过滤元组。
- **步骤 C：选币逻辑优化 (Select Logic)**：
  - **状态**：**（目前留空）**。
- **步骤 D：落盘保存**：
  - **输出**：将主观判断与优化方案保存至输入的单币持仓段目录下，文件名为 `ai分析以及优化方案.md`。
